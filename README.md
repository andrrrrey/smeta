# SMETA — Telegram-бот для расчёта смет и коммерческих предложений

Telegram-бот, автоматизирующий расчёт стоимости монтажных работ и материалов по спецификациям оборудования. Бот извлекает позиции из PDF-файлов, ищет цены во внутренней базе, векторной БД (RAG) и интернете через OpenAI, после чего формирует коммерческое предложение в виде Excel-файла.

## Возможности

### Пользователи

- **Загрузка спецификации** — отправьте PDF с перечнем оборудования. Бот автоматически определит диапазон страниц или позволит указать его вручную.
- **Расчёт цен** по трём источникам (в порядке приоритета):
  1. Внутренний прайс-лист (нечёткое совпадение, порог 70%)
  2. Векторная база знаний (ChromaDB RAG)
  3. Интернет-поиск через OpenAI GPT
- **Цветовая маркировка в Excel**:
  - Зелёный — цена из внутреннего прайса
  - Жёлтый — цена из интернета/ИИ
  - Красный — цена не найдена
- **История расчётов** — просмотр, редактирование и удаление сохранённых смет.
- **Редактирование расчётов** — текстовые команды, голосовые сообщения или загрузка исправленного Excel-файла.
- **Подтверждение расчёта** — после финализации смета становится неизменяемой.

### Администраторы

- Загрузка и сброс внутреннего прайс-листа (CSV)
- Управление стоп-словами (расходники, у которых интернет-поиск пропускается)
- Управление заголовками разделов спецификации
- Управление базой знаний (RAG): загрузка TXT/PDF/XLSX, удаление по источнику
- Настройки ИИ: смена модели, редактирование системного промпта, обновление API-ключа
- Просмотр списка пользователей, блокировка/разблокировка

## Архитектура

```
Telegram Bot (Aiogram 3)
        │
        ▼
  handlers.py          ← Обработчики сообщений, FSM-состояния
        │
        ▼
  services.py          ← AIService, VectorDB, PriceLogic
  ├── AIService         ← Запросы к OpenAI (цены, распознавание, транскрипция)
  ├── VectorDB          ← ChromaDB: добавление/поиск/удаление документов
  └── PriceLogic        ← Пайплайн расценки: прайс → RAG → интернет
        │
        ▼
   utils.py             ← Парсинг PDF, генерация Excel, очистка текста
        │
        ▼
    db.py               ← SQLAlchemy модели, инициализация БД
        │
        ▼
   config.py            ← Чтение config.ini, Settings-объект
```

**Поток обработки спецификации:**

```
PDF → Извлечение таблиц → Очистка позиций → Стриминг в пайплайн расценки
                                                      │
                                         Параллельное ценообразование (до 3 потоков):
                                           1. Внутренний прайс (fuzzy match)
                                           2. Векторная БД / RAG
                                           3. OpenAI (интернет)
                                                      │
                                         Сохранение в БД → Excel → Отправка пользователю
```

## Установка

### Требования

- Python 3.10+
- Токен Telegram-бота (получить у [@BotFather](https://t.me/BotFather))
- API-ключ OpenAI

### Клонирование и установка зависимостей

```bash
git clone <url-репозитория>
cd smeta
pip install -r requirements.txt
```

## Конфигурация

Отредактируйте файл `config.ini`:

```ini
[bot]
# Токен бота, полученный от @BotFather
token = ВАШ_ТОКЕН_БОТА

# ID пользователей Telegram с правами администратора (через запятую)
owner_ids = 123456789,987654321

[database]
# Путь к SQLite-базе данных
db_url = sqlite+aiosqlite:///main.db

[vector_db]
# Директория для хранения ChromaDB
path = ./chroma_db
# Название коллекции в ChromaDB
collection_name = internal_docs

[api]
# API-ключ OpenAI (можно также задать через панель администратора)
openai_api_key = ВАШ_КЛЮЧ_OPENAI
# Модель OpenAI по умолчанию
ai_model = gpt-4o
```

> **Примечание:** API-ключ OpenAI и модель можно изменить прямо в боте через меню администратора — изменения сохраняются в базе данных.

## Запуск

```bash
python main.py
```

При запуске бот:
1. Инициализирует SQLite-базу данных (создаёт таблицы / применяет миграции)
2. Загружает настройки, стоп-слова, заголовки разделов и кеш прайса
3. Запускает Telegram-поллинг

## Использование

### Для пользователей

**Команды:**
- `/start` — запуск бота, главное меню
- `/help` — справка по работе с ботом

**Главное меню:**

| Кнопка | Действие |
|--------|----------|
| 📊 Рассчитать спецификацию | Начать новый расчёт (загрузить PDF) |
| 📚 Мои расчёты | История расчётов |

**Процесс расчёта:**

1. Нажмите **📊 Рассчитать спецификацию**
2. Отправьте PDF-файл со спецификацией
3. Выберите режим определения страниц (авто или вручную)
4. Дождитесь обработки — бот сообщает о прогрессе
5. Получите Excel-файл с коммерческим предложением

**Редактирование расчёта:**

- Выберите расчёт из истории
- Отправьте текстовую или голосовую команду, например:
  - `"увеличь количество позиции 3 до 5"`
  - `"измени цену монтажа для вентилятора на 1500"`
  - `"добавь 10% к итогу"`
- Или загрузите отредактированный Excel-файл

После редактирования нажмите **✅ Одобрить** для финализации (одобренные расчёты нельзя изменить).

### Для администраторов

Доступ: `/admin` (только для пользователей из `owner_ids`).

| Раздел | Функции |
|--------|---------|
| 📋 Прайс-лист | Загрузить CSV (колонки: `Наименование`, `Цена`), очистить прайс |
| 🚫 Стоп-слова | Добавить/удалить слова-расходники (пропускают интернет-поиск) |
| 📑 Разделы | Управление заголовками разделов спецификации |
| 🔴 Расходники | Список расходных материалов |
| 🤖 Настройки ИИ | Смена модели, системный промпт, API-ключ |
| 🧠 RAG / База знаний | Загрузка документов (TXT/PDF/XLSX), удаление по источнику |
| 👥 Пользователи | Просмотр, блокировка/разблокировка |

**Формат CSV для прайс-листа:**

```csv
Наименование,Цена
Монтаж вентилятора,1500
Прокладка воздуховода (м),350
```

Поддерживаются альтернативные названия колонок: `наименование`, `name`, `название` для наименования и `цена`, `price`, `стоимость` для цены.

## База данных

SQLite с асинхронным доступом через SQLAlchemy. WAL-режим включён для улучшения конкурентности.

### Модели

| Таблица | Назначение | Ключевые поля |
|---------|-----------|---------------|
| `users` | Telegram-пользователи | `user_id`, `username`, `first_name`, `is_blocked`, `created_at` |
| `calculations` | Расчёты/сметы | `id`, `user_id`, `status` (pending/approved), `total_cost`, `pdf_filename`, `created_at` |
| `calculation_items` | Позиции расчёта | `id`, `calculation_id`, `name`, `code`, `quantity`, `unit`, `cost_per_unit`, `cost_material_per_unit`, `total_cost`, `source` |
| `price_list_items` | Внутренний прайс | `id`, `name` (unique), `price`, `price_material` |
| `stop_words` | Стоп-слова (расходники) | `id`, `word` (unique) |
| `consumable_words` | Расходные материалы | `id`, `word` (unique) |
| `section_titles` | Заголовки разделов | `id`, `title` (unique) |
| `bot_settings` | Настройки бота | `openai_api_key`, `ai_model`, `system_prompt` |

**Источники цен** (поле `source` в `calculation_items`):

| Значение | Описание |
|----------|----------|
| `internal` | Цена из внутреннего прайса |
| `internet` | Цена из интернета (OpenAI) |
| `rag` | Цена из векторной базы знаний |
| `consumable` | Позиция-расходник (цена не ищется) |
| `section` | Заголовок раздела (не тарифицируется) |

## Требования к PDF

- Таблицы со спецификацией должны содержать колонки с наименованием, единицами измерения и количеством
- Поддерживаются как текстовые PDF, так и сканы (для сканов используется Vision API)
- Рекомендуемый размер файла — до 20 МБ (ограничение Telegram)
